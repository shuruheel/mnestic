## 1. Why / what this buys

The engine has the two halves of budgeted expansion and not their intersection (verified 2026-07-10): `BFS`/`DFS` carry a global `limit`, but it counts *condition-satisfying* nodes, not visited nodes, they are unweighted, and they have **no depth bound of any kind** (`fixed_rule/algos/bfs.rs:89-91` — `found.len() >= limit` breaks the outer loop while traversal visits non-matching nodes without bound); `ShortestPathDijkstra` is weighted but takes **no budget** and computes the full single-source tree — with no termination input it emits *every node in the graph*, unreachable ones at `+inf` (`algos/shortest_path_dijkstra.rs:236-246`); `min_cost_k` caps per-group *derivations*, never distinct *nodes*. Anything that walks a graph to fill a fixed context window needs exactly the missing primitive.

The flagship consumer proves the gap twice over. mindgraph-rs's `traverse_reachable` reimplements level-by-level BFS in ~200 lines of host Rust (`mindgraph-rs/src/storage/cozo.rs:1748-1955`) because the semantics — global distinct-node budget, weighted costs, deterministic admission, an in-expansion liveness gate — cannot be expressed in a rule; and its retrieval pipeline (`/retrieve/context`) runs hybrid search, collects `matched_node_uids`, and then does **no multi-hop expansion from those seeds at all** — only one-hop edge lookups among already-matched nodes. The natural "fill the remaining context budget with the cheapest graph neighborhood around what recall found" call does not exist. The surveyed GraphRAG systems confirm the shape of the gap: Microsoft GraphRAG, Zep/Graphiti, LlamaIndex, and FalkorDB's GraphRAG SDK all do hop-bounded/shape-bounded candidate collection, then post-hoc score reranking, then token/count truncation — **none of the surveyed systems admits context by cost-ordered, globally-budgeted expansion** (§13). Zep's node-distance *reranker* is the nearest neighbor: it re-sorts an already-fetched candidate list by graph distance, where this rule makes distance drive admission itself.

Before (host language, or not at all):

```
// mindgraph-rs: ~200 lines of level-synchronized BFS in Rust, one batched
// engine query per level, budget/gate/tie-break logic all host-side.
```

After (one call, composable into any downstream rule):

```
context[node, cost, parent, depth] <~ BudgetedTraversal(
    *edge[fr, to, w], seeds[n], *live[uid, ok],
    admit: ok, max_nodes: 200, max_cost: 12.0)
?[node, cost] := context[node, cost, parent, depth], node != 'q'   // plain Datalog from here

?[node, cost, parent, depth] <~ BudgetedTraversal(graph: 'knows', seeds[n], max_nodes: 500)
```

(The gate relation `*live[uid, ok]` is shaped so the admitted column sits in the bound prefix; gate bindings are positional — see §3.1 — so a gated column buried deep in a wide relation needs a purpose-shaped view, not a bare `*node[uid, …, tomb]`.)

The result is the **settled fragment of the multi-source shortest-path tree**: the `max_nodes` cheapest distinct nodes reachable from the seed set, each with its cost, parent pointer, and depth — standard `distTo[]`/`edgeTo[]` semantics (Sedgewick & Wayne §4.4), not invented ones. Parent pointers reconstruct any path in plain Datalog without shipping every path. It is also, honestly named, **multi-seed graph k-nearest-neighbors with a cost ceiling and optional hop bound**: early-terminated multi-source Dijkstra. The "first k settled = k cheapest" property follows directly from the label-setting invariant (§3.2); it is the single-source case Har-Peled calls "probably folklore" (arXiv:1607.07818) and is stated explicitly for road networks in the incremental-network-expansion k-NN lineage (survey arXiv:1601.01549).

With `graph:` input the call consumes the cached projection, so the per-call cost is proportional to the **explored region** (≤ `max_nodes` settled nodes + their incident edges), not the graph — the D3 lesson (setup was ~86% of a whole-graph kernel call) applied to the one operation whose output is *deliberately* local. Positional input pays the standard one-scan CSR build like every other algo; §3.2 states that trade honestly. **Flagship caveat (load-bearing, not hidden):** the projection speedup presumes an edge relation whose weight column *is* the traversal cost. MindGraph's costs are `−ln(clamp(weight))` with edge-tombstone filtering, which a raw `*edge` relation does not carry — so to get the `graph:`-projection locality the flagship must maintain a projectable derived cost relation `{from, to => cost}` (the `−ln` and liveness baked in at write time, a write-amplification cost) or accept the positional `O(live edges)` CSR build per call. This is the §3.5 lazy-positional trigger and the §10 oracle's edge-input contract; the round-trip reduction below is real either way, but the projection-locality win for the flagship specifically is gated on that maintained relation.

Expected effect (labeled expectation, not measurement): for MindGraph's `/traverse` surface, one engine call replaces ~2·depth `run_script` round-trips per traversal; for `/retrieve/context`, a capability that currently does not exist. This presumes the flagship enables the `graph-algo` feature (off in mindgraph-rs today — §10 step 1) and, for the projection arm, maintains the cost relation just described.

## 3. Design

### 3.1 Surface

Registered name **`BudgetedTraversal`** (builtin table, `fixed_rule/mod.rs:979+`; no alias in v1). `supports_projection() -> true`. Feature `graph-algo`. Arity **4**: `[node, cost, parent, depth]`.

Inputs (after the `graph_input` shift — positional edges: seeds at index 1, gate at 2; `graph:`: seeds at 0, gate at 1):

- **edges** — positional `[from, to, weight?]` (missing weight column ⇒ 1.0) **or** `graph: 'G'`. Built/consumed via `graph_input(0, VariantSpec::weighted(undirected, strict = true), &poison)` — negative, NaN, or infinite weights are loud errors, zero weights are legal (§3.2). Projections refuse TxTime sources (graph-projection.md §3.1), so temporal edge relations feed the positional form — that asymmetry is inherited, not introduced.
- **seeds** — required; column 0 is the node id, optional column 1 is `initial_cost`; **columns 2+ are ignored** (documented, so a wide relation passed as seeds is not a silent surprise — but its column 1, whatever it means, *is* read as `initial_cost` if numeric, so pass a purpose-shaped seed relation). Full `iter()` scan (seed sets are small by construction). `initial_cost` must be a finite number ≥ 0 (loud error otherwise); missing ⇒ 0.0. A non-numeric column-1 value is a loud error, not a silent skip. Duplicate seed rows: the minimum initial cost wins. A seed whose `initial_cost` exceeds `max_cost` is **inadmissible** — under the super-source framing (§3.2) its path costs `initial_cost > max_cost`, so by §4's ≤`max_cost` rule it spends no budget, expands nothing, and emits nothing (the seed-boundary counterpart of a reached node over the ceiling). A seed absent from the CSR is **not** silently skipped (contra the Dijkstra precedent, deliberately — §5): it is gated, admitted, and emitted like any node; it simply has no out-edges to expand.
- **gate** — optional; a relation whose **first column is the node id** and whose subsequent columns feed `admit:`. When present, a node (seed or reached) is *admissible* iff `prefix_iter(node)` yields a row satisfying `admit:` (below); when absent, every node is admissible. The quantifier is **existential over the node's gate rows** (amendment A4): admissible iff *some* probed row satisfies `admit:` — order-independent, and a deliberate strengthening of the BFS `condition` pattern this copies (`algos/bfs.rs:74-87` evaluates only the first probed row). Probed lazily per candidate — stored gate relations are never materialized (`fixed_rule/mod.rs:118-145`). **Positional-prefix caveat:** the bindings bind a *prefix* of the gate relation's columns (arity is only checked ≥ the binding count), so an `admit:` referencing a column that lands at the wrong position (e.g. a tombstone flag deep in a wide `*node`) silently gates every candidate out — shape the gate relation, or a view, so the gated column sits in the bound prefix (the §11(j) misbinding fixture pins this).

Options (validated via the payload helpers, so diagnostics ride the standard `fixed_rule::arg_not_found` / `WrongFixedRuleOptionError` errors):

| option | type | default | meaning |
|---|---|---|---|
| `max_nodes` | positive integer | **required — no default** | global budget of distinct admitted nodes, **seeds included**. Required deliberately: the caller must think about the budget (the antichain "no default on the cap" doctrine); an accidental unbounded expansion is exactly the failure mode this rule exists to prevent. |
| `max_cost` | **finite** float ≥ 0 | unset = unbounded | admissible-path cost ceiling (NetworkX `cutoff` semantics: only nodes with min-cost ≤ `max_cost` are admitted, **seeds included** — §3.1 seeds bullet). NaN and ±inf are **loud errors** (amendment A3): NaN would silently never prune (`c' > NaN` is false), and "unbounded" is spelled exactly one way — omit the option. |
| `max_depth` | positive integer | unset = unbounded | admissible-path hop ceiling — **exact** semantics via layered labels when weighted (§8) |
| `undirected` | bool | `false` | traverse both orientations (CSR symmetrized at build; a cached `(undirected, weighted)` variant, no new cache key) |
| `admit` | expression | unset = bare presence | evaluated against each probed gate row (binding map from the gate input's bindings, the BFS `condition` pattern); setting `admit:` with no gate input is a **loud error** (a bug, not a no-op — an `admit:` is meaningless without a gate relation) |

### 3.2 The core algorithm

Multi-source Dijkstra (equivalently: uniform-cost search from a virtual super-source whose edge to seed *s* costs `initial_cost(s)` — the two are the same algorithm, so the spec defines semantics by the super-source and implements without materializing it), early-terminated at `max_nodes` admissions.

```
frontier: binary heap ordered by (cost, node-DataValue [, hops])   // §3.3
labels:   per node (unbounded depth) or per (node, hops) (§8)
admitted: count; emitted rows buffered until termination

for each gated seed s with initial_cost(s) ≤ max_cost:
    push (initial_cost(s), s, hops=0, parent=Null)     // seed over the ceiling is inadmissible (§3.1)
loop (poison mask every 4096 pops+relaxations):
    pop heap entry (c, v, h, p)
    if v (or, layered, state (v,h)) is already settled → continue   // entry used only as a (cost,node[,hops]) key
    settle v (layered: state (v,h)) from its current stored best label
    if v not yet admitted:
        admit v; emit (v, c, parent-of-best, h); admitted += 1
        if admitted == max_nodes → stop
    for each out-neighbor (v → u, w) in the CSR:            // undirected already symmetrized at build (§3.1);
        c' = c + w (f64);  h' = h + 1                       // out-neighbors carry both orientations
        if c' > max_cost or h' > max_depth → prune
        if u not admissible (gate probe, memoized) → prune   // gate: zero budget, no bridge (§7)
        relax to u if it strictly-lexicographically improves u's label (§3.3)
emit buffered rows; RegularTempStore canonicalizes order
```

The pop's settle test compares the popped node/state against the *stored* best (a stale heap entry — one whose `(cost, parent)` was since improved — is discarded), so the emitted parent is always the stored best, never whichever duplicate happened to pop first; this is what makes the witness form-independent (§3.3), and §11's discrimination plan flips it (settle from the popped payload instead of the stored label → the form-equivalence row goes red).

Correctness rides the **label-setting property**: with non-negative weights (and non-negative initial costs), a node's first settled label carries its final min cost, so nodes settle in nondecreasing cost order. **Under strictly positive weights** the first `max_nodes` distinct settles are exactly the `max_nodes` cheapest nodes, tie-broken by node DataValue — the plain sorted prefix (§3.3). **Zero-weight edges are legal** (the super-source construction is zero-weight edges; only negative weights break the invariant) but make a node reachable *through* an equal-cost node, so the admitted set is the settle-order prefix, not the plain sorted prefix — §3.3 gives the exact declarative characterization and the §11 zero-weight-chain fixture pins the difference.

Complexity, honestly stated: unbounded depth, `O(m_B log n_B)` where `n_B ≤ max_nodes` settled nodes and `m_B` their incident edges — **independent of graph size** on the `graph:` arm; positional input adds the standard one-scan CSR build every other algo pays. Depth-bounded weighted runs multiply label state by up to `max_depth + 1` (§8); this is conditionally optimal, not an implementation weakness (Bellman-Ford's `O(D·m)` is tight under the Min-Plus-Convolution hypothesis, and under APSP for `h ∈ O(√m)`; arXiv:2211.07325). Memory is bounded by labels + frontier, i.e. by the explored region, not the graph.

What v1 does NOT need and why it cannot arise: incremental maintenance under edge deletion (the rule runs inside one transaction against one snapshot; there is no cross-call state), and re-expansion of settled labels (label-setting; the strict-improvement relaxation discards non-improving paths).

### 3.3 Determinism: total-order admission, canonical output

The budget is a cut, and ties at the cut boundary decide which nodes *exist* in the output — so admission order must be a total order or the rule violates the house cap dichotomy (deterministic-by-total-order or loud error, never a silent partial-order truncate). The declarative contract, stated without reference to any queue:

- **Settle order** is the sequence produced by iterating: *among all admissible nodes whose min-cost admissible path uses only already-settled nodes as intermediates, select the `(mincost, node-DataValue)`-smallest.* This is a total order (mincost is a function of the node; DataValue order is total), and it is exactly what label-setting Dijkstra with the `(cost, DataValue)` comparator produces. **Under strictly positive weights it equals the plain `(mincost, DataValue)`-sorted order** (every min-cost predecessor of a cost-`c` node has cost `< c`, so all cost-`c` nodes are settleable before the first is settled); under zero-weight edges the "uses only already-settled intermediates" clause is what keeps it well-defined and prefix-closed (a node cannot settle before the equal-cost node its only path runs through).
- **Admitted set** = the length-`max_nodes` prefix of the settle order (so it is parent-closed by construction — a tree fragment, matching §1's SPT-fragment framing).
- **Emitted row** for admitted `v` = `(v, mincost(v), parent, depth)`, where `(depth, parent)` is the witness defined below.

Implementation discipline that realizes it: the heap comparator is `cost.total_cmp` then a comparison of the node's **DataValue** — `indices[id]` for a CSR node, the raw seed value for a CSR-absent seed (then `hops` ascending in layered mode). **DataValues, never raw u32 ids**: CSR ids are assigned in interning order and differ between a positional build and a projection with a `nodes:` slot (`fixed_rule/mod.rs:435-468`), so a u32 tie-break would make output depend on input *form*. Seed↔CSR matching is likewise structural (DataValue identity), the fork's known Int/Float seam (§7 precision note). This is the `min_cost_k` discipline ("ties order by the whole pack") applied to a frontier, and the composition is safe per the antichain §3.4 recipe: `total_cmp` on cost is a strict weak order, DataValue order is total — lexicographic composition is total. **Witness (parent, depth):** relaxation replaces `v`'s label only on strict lexicographic improvement of `(cost, hops, parent-DataValue)`, so among equal-cost predecessors the smallest `(hops, parent)` wins independently of neighbor-iteration order (the 802.1aq lowest-id principle, adopted by that standard precisely to make independently-computed equal-cost trees identical).

*Precision note (scoping the guarantee, per house rule):* the admitted **set** and each node's **cost** are exact declarative functions of (relation state, parameters) — schedule- and form-independent, pinned by permuted-input and positional-vs-`graph:` equivalence tests (§11 d, e), **including under zero-weight edges** (the settle-order definition above is the pinned contract, and the zero-weight-chain fixture, §11, checks it against an independent implementation of that definition). The `(depth, parent)` **witness** is deterministic (same inputs ⇒ same output, pinned by the same tests) and is the lexicographic minimum in the common case, but under **zero-weight edges from equal-cost predecessors** (a cycle is merely one instance) a node settled early does not retroactively adopt a lexicographically-smaller parent discovered via an equal-cost path through a later-settled node — the witness is canonical-among-explored, not canonical-among-all-paths. Full path-ID canonicalization (802.1aq's actual mechanism) is rejected for v1: it costs path-vector comparisons per relaxation to canonicalize a field no known consumer reads canonically. Documented, not hidden.

*Amendment A1 (2026-07-11) — the cross-mode face of the same scoping.* "Canonical-among-explored" also means the witness is relative to **which labels the mode explores**. Layered mode (§8) deliberately keeps states that single-label mode discards (an equal-cost, fewer-hops arrival at an already-settled node — Pareto-incomparable, and load-bearing for exactness when the depth bound binds). On a **zero-weight plateau** such a state can only be discovered *after* the node settles, so the two modes explore different label sets and can emit different — each individually deterministic, each canonical-among-its-own-explored — `(parent, depth)` witnesses for a downstream node. The admitted **set** and every **cost** are identical regardless. This is what scopes §8's byte-equality claim; it is a consequence of §3.2's settle-once rule and §8's exactness obligation being jointly binding, not a defect in either.

### 3.4 Guards, laws, errors

The guards divide the labor: **`max_nodes` backstops unbounded admission** (required, no default); **`max_cost`/`max_depth` bound the admissible path space** (optional, semantic); **poison backstops wall-clock** (`::kill` / `:timeout`, every-4096 mask — the new loop must not copy `dijkstra()`'s zero-check gap). Caller obligations, all loudly enforced: weights non-negative and finite (strict `VariantSpec`; positional builds fail during the scan, cached variants at consume time); seed `initial_cost` finite and ≥ 0 (non-numeric column 1 is an error); `max_cost`, when given, finite and ≥ 0 (NaN and ±inf are errors — A3); `max_nodes ≥ 1`; `admit:` without a gate input is an error.

*Zero-sign canonicalization (amendment A4):* seed `initial_cost` and `max_cost` normalize `-0.0` to `+0.0` on entry, so a single convention — `total_cmp` for **every** cost comparison (heap order, stale check, improvement, dominance) — holds throughout the run and no emitted byte can split on zero's sign. (CSR weights cannot reintroduce it: IEEE `+0.0 + -0.0 = +0.0`.) The `admit` expression must be pure (same contract as BFS's `condition`); a throwing expression aborts the rule with its own error. Empty-graph guard: `indices().is_empty()`, never `node_count() == 0`; gated seeds still emit on an empty graph (a seed's existence does not depend on it having edges).

### 3.5 What is deliberately NOT in v1

Per-seed attribution column (derivable by walking parent pointers; a `seed` column would misread as per-seed expansion semantics — rejected in §5); **lazy positional adjacency** via `prefix_iter` instead of the eager CSR build (trigger: *a measured consumer whose per-call scan+build cost exceeds its explored-region cost and whose edge relation cannot be projected* — the known candidates are TxTime edge relations, which projections refuse, and the flagship's `−ln`-cost/tombstone-filtered edge shape until it maintains a projectable cost relation, §1; the out-of-tree prototype (§10) is where that measurement happens, baseline-first); **edge-payload emission** — edge type, per-edge confidence: the flagship's node-hydration join for `label`/`node_type` is kept regardless (§7), but this deferral is *not* free on top of it — recovering the wire's `edge_type` forces a separate `(parent, node)` edge join that must reproduce the parallel-edge collapse-to-strongest (`cozo.rs:1791-1809`, not a plain join), and the wire's `path_confidence` needs a per-edge confidence fold (`cozo.rs:1898`) along the parent chain on top of that join; the engine's single `cost` column substitutes only for `path_cost`. Trigger: a consumer for whom that parent-pair join or the confidence fold is the measured bottleneck. Also deferred: a hop-major compatibility mode reproducing `traverse_reachable`'s level-ordered admission (the oracle defines an equivalence class instead, §10; trigger: a consumer that cannot migrate semantics); path canonicalization under zero-weight edges (§3.3 precision note); k-shortest-paths, per-seed trees, PPR/decay scoring (different primitives — `KShortestPathYen` and future work respectively); any HybridSearch code (§9 is a seam statement, its own build item).

## 4. The budget decision (Q1 — resolved: `max_nodes` required + `max_cost` optional, deterministic truncation)

Engine-internal argument: the two shipped cap precedents are `min_cost_k` (deterministic-by-total-order truncation) and the divergence epoch cap (loud error). A context-fill budget is definitionally a truncation — erroring on "graph bigger than budget" would defeat the primitive — so it must be the deterministic-by-total-order side, which §3.3 supplies. The literature agrees from the other direction: early-terminated Dijkstra after k settles is *the* textbook mechanism for "k nearest in a graph" (road-network k-NN, incremental network expansion), and its determinism gap (no surveyed system documents tie-breaking, §13) is exactly what a spec-level total order closes. `max_cost` composes freely (a pure prune on a monotone frontier — NetworkX `cutoff`), costs nothing when unset, and serves the retrieval consumer ("relevance radius") — both knobs, `max_nodes` required, `max_cost` optional. **Decision: as stated.** _Rejected:_ `max_nodes` optional-with-default (an accidental full-graph expansion is this rule's own anti-goal; the antichain "no default on the cap" doctrine); budget = cost-ceiling-only (fails the context-window use case — token budgets count items, not path costs); budget counting *expansions* rather than admissions (Memgraph's `USING HOPS LIMIT` counts work units and returns arbitrary partial results — the anti-pattern: a resource guard, not a semantics).

*Precision note:* seeds count toward `max_nodes` (they occupy the same context window), so `max_nodes < |seeds|` admits the cheapest seeds by `(mincost, node)` order — consistent, no special case. A seed with `initial_cost > max_cost` is not admitted at all (§3.1), the seed-boundary counterpart of the cost ceiling.

## 5. The output decision (Q2 — resolved: `(node, cost, parent, depth)`, seed roots at depth 0, no seed column)

`(node, cost, parent, depth)` is the settled SPT fragment in standard `distTo`/`edgeTo` form: parent pointers are cheaper than paths, reconstruct any path in plain Datalog, and compose (downstream rules join on `parent` directly). A seed emits `(seed, mincost, parent, depth)` where `mincost = min(initial_cost, cheapest admissible path cost from the seed set)` — a seed reachable from *another* seed more cheaply than its own `initial_cost` takes that cheaper cost and that path's `(parent, depth)`. It emits the root form `(seed, initial_cost, Null, 0)` iff `initial_cost` is that minimum; the tie rule falls out of §3.3's lexicographic relaxation for free (the root label has `hops = 0`, so on a cost tie the `(Null, 0)` root wins over any equal-cost path). Cost emits as f64. One row per node, guaranteed by the rule (the output store would happily keep two rows differing in parent — `runtime/temp_store.rs:27-61` — so uniqueness is the loop's invariant, tested in §11).

**Depth-semantics note (wire-visible):** the emitted `depth` is the hop count of the min-cost witness path, which for a weighted graph can exceed the fewest-hops depth. This differs from mindgraph-rs's `PathStep.depth`, documented today as BFS fewest-hops discovery depth (`traversal.rs:62-95`) — a migrating consumer must expect that field's meaning to change (it is the hop-major-compatibility deferral, §3.5/§10).

**Decision: as stated.** _Rejected:_ a fifth `seed` attribution column (the external analysis's `(seed, node, …)` shape) — under merged-frontier semantics the "owning" seed is just the root of the parent chain, derivable in three lines of Datalog; a dedicated column would either be redundant or, worse, imply per-seed expansion (k trees, a different and more expensive primitive); path-list emission (`ShortestPathDijkstra` emits full path lists — at `max_nodes` = 500 that is 500 materialized lists nobody asked for); silent-skip of unknown seeds (the Dijkstra precedent, `algos/shortest_path_dijkstra.rs:53-59`) — a context-fill caller cares about every seed, and the gate (§7) is the principled place to drop dead ones; edge-type/confidence columns (§3.5, deferred with trigger).

## 6. New rule vs extending `BFS` (Q3 — resolved: a new FixedRule)

`BFS`'s `limit` already means "stop after N *condition matches*" (`algos/bfs.rs:89-91`) — overloading it with "budget of admitted nodes" would silently change shipped semantics, exactly the never-silently-break-upstream rule. `BFS` is also structurally excluded from `graph:` (its per-tuple `condition` reads relation tuples a CSR does not carry — graph-projection.md §3.5.4 names the lazy family and designates this rule as "their cache-consuming successor"). A new rule keeps `BFS` byte-compatible, gets projections from day one, and — as a FixedRule — is a stratification boundary, so no fixpoint-confluence question arises (fixed rules run once per evaluation, `query/eval.rs:216-226`). **Decision: new FixedRule `BudgetedTraversal`.** _Rejected:_ new options on `BFS` (semantic collision on `limit`; no projection path); a magic recursive-rule pattern the planner recognizes (evaluator surgery, the anti-overbuild line); naming it `KNearestNodes` (collides with the engine's vector-k-NN vocabulary) or `BudgetedExpansion` (the roadmap's public noun is *traversal*; keep the vocabulary aligned).

## 7. The liveness gate (Q4 — resolved: the generic mechanism moves in-engine; the policy stays the caller's)

The roadmap asked whether the rule subsumes `traverse_reachable`'s liveness gate or leaves it a host post-filter. The consumer code answers the second half definitively: **a post-filter cannot work.** mindgraph-rs's f28c496 exists because gating after expansion let dead nodes (a) consume `max_nodes` budget and (b) act as bridges (`A → tombstoned → B` wrongly reaching B) — and the deterministic cheapest-first fix (ff55482) made the starvation *reliable* (a strong-edged dead node deterministically won a budget slot). Whoever owns the budget must own the gate.

What moves in-engine is only the mechanism, which passes the stranger test cleanly: *"admit only candidates that appear in relation R (first column = candidate) with a row satisfying predicate P"* — no cognitive vocabulary. The shipped scaffolding already provides both halves: lazy keyed probes into stored relations (`prefix_iter`, `fixed_rule/mod.rs:118-145` — O(frontier) probes, no materialization, works identically alongside `graph:` because the gate is a separate input from the edges) and per-tuple expression evaluation against a relation row (BFS's `condition`, `algos/bfs.rs:34-41` setup + `:70-92` per-row probe/eval). The tombstone *sentinel* (`tombstone_at == 0.0`, a Float convention) stays exactly where it belongs — in the `admit:` expression the caller writes. Gate semantics: an inadmissible node spends zero budget, emits nothing, and is never expanded (not a bridge); the gate applies to seeds too; a dangling edge target (edge to a node with no gate row) fails bare presence — which makes the gate also the principled answer to "typo'd seed" (§3.1). Probe results are memoized per call (a node is probed once, not once per incident edge). **Decision: optional gate input + `admit:` expression, as §3.1.** _Rejected:_ host post-filter (breaks budget and bridge semantics — proven in production by f28c496); presence-only gate without `admit:` (tombstoned nodes *have* rows; presence cannot express the flagship's gate); a materialized pre-filtered inline-rule input (inline rule args materialize into an epoch store — O(all live nodes) per call, defeating a budgeted traversal; measured shape, `fixed_rule/mod.rs:92-98`); `(gate_column, gate_value)` option pairs (narrower than the shipped expression machinery it would sit beside, and dies on the first two-condition gate).

*Precision note:* the engine `==` in an `admit:` expression compares numerically (`1 == 1.0` is true) while gate-relation *probing* keys on DataValue identity (`1` ≠ `1.0`) — the fork's known Int/Float seam, and the same seam governs seed↔CSR matching (§3.3). For the flagship this is moot (its sentinel is uniformly Float and its ids uniform), but the rustdoc must state it; the mixed-type fixture is §11(j).

## 8. The depth decision (Q5 — new in this spec; resolved: exact, via layered labels; never depth-pruned single-label Dijkstra)

The roadmap's language ("cheapest-first … and a depth bound") reads as if the two compose trivially. They do not — this is the one place the grounding research overturned an assumption. With weights, single-label Dijkstra with depth pruning is **wrong**, not merely approximate-with-caveats: a node whose globally-cheapest path exceeds `max_depth` gets settled at that infeasible cost (or pruned entirely) while a costlier-but-shallower path was admissible — the classic "cheapest flights within K stops" trap, and its output would depend on exploration order, which is precisely the "leaking evaluation scheduling into results" the house confluence doctrine forbids.

The exact semantics is declarative: `mincost_D(v)` = minimum cost over admissible paths of ≤ `max_depth` hops. The construction is the layered graph (states `(v, i)`, `i ≤ D`), realized as **exactly one label per `(node, hops)` state**, with two distinct disciplines:

- **Within a state**, the `(node, hops)` label is replaced only on strict lexicographic improvement of `(cost, parent-DataValue)` — the §3.3 relaxation rule with `hops` fixed by the state key. So the surviving parent is the minimum parent-DataValue independent of CSR neighbor order (**never** raw weak-Pareto keep-first, which would make the parent form-dependent — the trap the review caught).
- **Across states** of the same node, pruning is **strict `(cost, hops)` Pareto dominance**: a state is dropped only when another state of the same node has `≤` cost *and* `≤` hops with at least one strict `<`. Two equal-`(cost, hops)` states never mutually prune (they are the same state), so there is no keep-first hazard.

Label-setting holds per `(node, hops)` state (hop count is a unit-consumption resource; the RCSPP label-setting lineage). A node is admitted and emitted on its **first settled label = the lexicographic minimum `(cost, hops, parent-DataValue)`** over its feasible states — min cost, then fewest hops (the heap's `(cost, node, hops)` order), then min parent; later labels of an admitted node keep relaxing (they can open cheaper feasible paths to *other* nodes) without emitting or re-counting. Cost: up to `max_depth + 1` labels per node — `O(D·m)` worst case, conditionally optimal (arXiv:2211.07325), and in practice bounded by the explored region like everything else here.

When `max_depth` is unset: plain single-label Dijkstra, zero overhead. When the graph is unweighted (all weights 1.0): cost ≡ hops, the layers collapse, and plain Dijkstra with depth pruning *is* exact — a legitimate fast path, an implementation note rather than a semantics fork.

**Non-binding `max_depth` ≡ unset — as amended (A1, 2026-07-11).** A `max_depth` set high enough not to bind produces:

- **the identical admitted set and identical `(node, cost)` columns — unconditionally.** Cross-node settle order is keyed `(cost, node-DataValue)` (hops is only a tie-break *within* one node's states), and per-state label-setting gives every node the same `mincost` when the bound does not bind, so each node's first settle occupies the same position in the same total order. Budget cuts mid-plateau included.
- **byte-identical full rows — under strictly positive weights.** No extra state layered mode explores can touch an emitted witness: an extra state of `v` costs `c₂ > mincost(v)`, so its relaxation along `(v,u,w)` costs `c₂ + w > mincost(v) + w ≥ mincost(u)`, and equality is impossible (it would force `mincost(u) > mincost(u)`). Min-cost arrivals therefore come only from min-cost predecessor states, which both modes generate identically and at identical heap times; and with strictly positive weights every equal-cost arrival lands *before* its node settles, so both modes choose from the same arrival set.
- **possibly divergent `(parent, depth)` on zero-weight plateaus.** The original spec claimed byte-equality unconditionally; that is **false**, and jointly unsatisfiable with §3.2's settle-once rule (proven counterexample: `a→m1 0, m1→m2 0, m2→b 0, a→z 0, z→b 0, b→t 0`, seed `a` — single mode settles `b` at `(0, m2, 3)` and must ignore `z`'s later equal-cost, fewer-hops arrival; layered mode must keep and expand that state, because when the bound *does* bind it can be the only feasible route onward — so `t` emits depth 4 in one mode and 3 in the other, at identical cost). No local rule closes this without breaking something signed: suppressing the late state breaks depth exactness; re-expanding in single mode contradicts §3.2's per-node labels and this section's "zero overhead". The divergence is the cross-mode face of §3.3's canonical-among-explored scoping, and is documented there. §11(g)'s equivalence fixture is therefore pinned on **strictly positive weights**, where the equality above is proven.

**Decision: exact layered semantics when `max_depth` is set on a weighted graph; the unweighted collapse as an optimization; depth-pruned single-label on weighted graphs is forbidden, including as a fallback.** _Rejected:_ documented-approximate depth pruning (procedural semantics; schedule-dependent output values; the antichain doctrine has no "documented" exemption for silent-wrong); deferring `max_depth` entirely (the flagship's traversal has carried a hop bound since inception — defaults 10 library / 5 HTTP — and `GraphLeg` is hop-bounded; shipping without it strands both consumers).

## 9. The HybridSearch expansion seam (stated, not solved — its own build item in 0.12)

The roadmap scopes 0.12 as the FixedRule **plus** its retrieval integration; this section pins the seam so the two halves stay separable. `HybridSearch` is a script *builder* (`runtime/hybrid.rs` module doc): its legs are generated Datalog rules fused by `ReciprocalRankFusion`. Today's `GraphLeg` (`hybrid.rs:109-140`) generates a recursive bounded min-hop rule from **explicit, caller-supplied seeds** — unweighted, hop-ranked, no budget. The integration replaces that generated recursion with one generated `BudgetedTraversal` call whose **seeds input is the vector/FTS legs' own top-k rule** (a tiny inline relation — the O(k) case where inline-input materialization is free), ranked into the fusion by ascending cost; `detailed: true` rows carry cost as the leg score, and parent pointers become recoverable per-leg detail. Two design questions deliberately left to that item, with the FixedRule's answer already compatible either way: whether seed `initial_cost` is derived from leg ranks (the seeds input's optional second column exists for exactly this) or left at 0; and whether `GraphLeg` is deprecated or kept as the no-budget fast path. What this spec guarantees the seam: FixedRules run in read-only scripts (mindgraph-rs already runs `ReciprocalRankFusion` under `ScriptMutability::Immutable` in production), and the seeds-from-a-rule shape needs no engine change — rule inputs are already first-class FixedRule inputs.

- (o) **Read-only**: runs under an immutable script (the §9/production RRF precedent).

## 12. Decisions (signed off by the owner 2026-07-11)

1. **Name `BudgetedTraversal`, new FixedRule, `graph-algo` feature, arity 4** (§3.1, §6). _Rejected:_ extending `BFS` (semantic collision on `limit`, no projection path); `KNearestNodes` (vector-k-NN vocabulary collision).
2. **Budget: `max_nodes` required with no default, seeds included; optional `max_cost` (seeds included)** (§4). _Rejected:_ defaulted budget; cost-only budget; work-unit budget (Memgraph `HOPS LIMIT` anti-pattern).
3. **Output `(node, cost, parent, depth)`; seed roots at `(initial_cost, Null, 0)`, but a seed reachable more cheaply takes that path; unknown seeds emit rather than silently skip** (§5). _Rejected:_ seed attribution column; path-list emission; Dijkstra's silent seed skip.
4. **Gate = optional relation input + `admit:` expression; in-expansion; zero budget, no bridging; sentinel stays caller vocabulary; positional-prefix binding documented as a footgun** (§7, §3.1). _Rejected:_ host post-filter (f28c496); presence-only; materialized live-view input; `(col, value)` option pairs.
5. **`max_depth` is exact via layered labels (within-state min-parent, across-state strict Pareto); unweighted collapse as fast path; depth-pruned weighted single-label forbidden** (§8). _Rejected:_ documented-approximate pruning; deferring depth.
6. **Costs accumulate in f64** (CSR weights stay f32, widened per relaxation — parity with `min_cost_k` and the lazy A* path, contra `ShortestPathDijkstra`'s f32 accumulation). _Rejected:_ f32 accumulation (precision drift on long cheap chains for zero savings — the CSR, not the accumulator, is the memory).
7. **Seeds may carry an optional non-negative `initial_cost` (column 2, default 0)** — the super-source construction generalized one step; ~15 lines. Kept for the honest reason: it is the natural super-source generalization, and while the §9 HybridSearch integration is a known 0.12 item, §9 explicitly leaves open whether it derives seed `initial_cost` from leg ranks or leaves it at 0 — so no consumer is yet *known to need* it. Deferral would not break anything (an optional trailing column is additive — the shipped `[from, to, weight?]` pattern; v1 would reject arity-2 seeds and a later relaxation to `[node, initial_cost?]` is purely additive). It is kept now, rather than deferred with a trigger, only because the ~15 lines land on the out-of-tree prototype we are writing anyway and pre-empt even that additive revision. _Rejected:_ negative initial costs (breaks the ≥0 uniformity of the label-setting argument for no consumer).
8. **Sequencing: `graph-algo` prerequisite + out-of-tree prototype (gate hardcoded, registration removed on 0.12 adoption) + DQP equivalence class (§10) before the in-tree decision; release vehicle 0.12.0 minor by roadmap headline designation; CHANGELOG-FORK in the same PR.** Contributor-suitability: the core loop is maintainer-led (novel algorithm + determinism obligations); the test matrix rows and the mindgraph-rs oracle harness are contributor-suitable.

### 12.1 Amendments ratified 2026-07-11 (post-sign-off, pre-build)

An adversarial review of the implementation design (both the core-loop and test-harness designs, reviewed against this spec and the engine source) proved one defect and three under-specifications. All four are ratified by the owner and applied above; each is marked in place.

- **A1 (defect).** §8's unconditional "non-binding `max_depth` ⇒ byte-identical output to unset" is **false** and jointly unsatisfiable with §3.2's settle-once rule on zero-weight plateaus (counterexample in §8). Amended: unconditional for the admitted set and `(node, cost)`; full-row byte-equality **under strictly positive weights**; the zero-weight witness divergence folded into §3.3's canonical-among-explored note. §11(g)'s fixture is pinned on positive weights.
- **A2 (defect in the test plan).** §11's discrimination plan predicted mutations 2 and 5 would redden the form-equivalence row (e). They redden nothing — no fixture in (a)–(o) exercises an equal-cost *improving second arrival*. Amended: new row (w1)–(w3) supplies the discriminators, mutation 1 needs interning skew in (c)/(e), and mutation 3's prediction is narrowed (depth-pruning is exact on unweighted graphs, so the unweighted row stays green). Reasoning recorded in §11.
- **A3 (strengthening).** `max_cost` must be **finite**: NaN and ±inf are loud errors, not "unbounded". NaN would silently never prune; and one spelling of unbounded (omit the option) keeps the surface honest. Rejecting is forward-compatible — accepting `+inf` now could not be revoked later. Probe added to §11(i).
- **A4 (clarifications).** Gate admissibility is **existential** over the candidate's gate rows (§3.1), a deliberate strengthening of the BFS `condition` pattern it copies. Seed `initial_cost` and `max_cost` canonicalize `-0.0` to `+0.0`, so one `total_cmp` convention governs every cost comparison in the run (§3.4).
- **A5 (amends decision 8 — sequencing; ratified 2026-07-11).** §10's out-of-tree **prototype is dropped**; the DQP oracle and the measurements survive unchanged, and move from *pre-build gate* to *pre-merge gate*, run from mindgraph-rs against the **in-tree builtin** through the existing path dependency (mindgraph-rs need only add `graph-algo` to its features — the rule then requires no registration). Rationale: the prototype is a throwaway reimplementation of the core loop whose hardcoded gate (§10's accepted fidelity gap) structurally *cannot* validate the generic `admit:` semantics — the design's highest-risk surface — while the in-tree §11 matrix is a strictly stronger correctness gate, and an oracle run against the builtin exercises the real gate rather than a stand-in. The engine cost of being wrong is low and bounded: the rule is additive, `graph-algo`-gated, and behind a new name, so an adverse measurement is a revert, not a migration. What the gate keeps, verbatim: **the mnestic PR does not merge until §10's oracle clauses (i)–(iv) are green against the builtin and the measurement table is recorded** (§10(3)'s numbers, and the §3.5 lazy-positional trigger answered with data). What is separately preserved: §10's never-exercised out-of-tree `supports_projection` → `graph:` path gets a dedicated in-tree test (a registered rule overriding `supports_projection`, called with `graph:`) rather than being validated incidentally by a prototype we then delete. _Rejected:_ dropping the oracle or the measurements (they are the gate's substance); keeping the prototype as well (duplicate core loop, lower-fidelity gate, scaffolding to delete on adoption).

## 13. Prior art

- E. W. Dijkstra — *A note on two problems in connexion with graphs* — 1959 — the label-setting invariant everything here rides on; Sedgewick & Wayne, *Algorithms* §4.4 — the `distTo`/`edgeTo` output convention adopted verbatim.
- S. Har-Peled — *Computing the k Nearest-Neighbors for all Vertices via Dijkstra* — arXiv:1607.07818 — states the single-source "first k settled = k cheapest" as (his word) "probably folklore"; cited here as adjacent support, the early-termination correctness resting on §3.2's own self-contained label-setting argument.
- Road-network k-NN lineage (Incremental Network Expansion; survey arXiv:1601.01549) — early-terminated Dijkstra after k dequeues as established practice — the `max_nodes` semantics with the determinism gap this spec closes.
- T. Kociumaka, A. Polak — *Bellman-Ford is optimal for shortest hop-bounded paths* — arXiv:2211.07325 — the §8 exactness cost is conditionally optimal (tight under Min-Plus Convolution; under APSP for `h ∈ O(√m)`); layered-graph reduction per MIT 6.006 solution sets.
- Desrochers & Soumis lineage — resource-constrained shortest paths, label-setting with dominance — hop count as the benign unit-consumption special case (§8's per-`(node, hops)` labels).
- IEEE 802.1aq / RFC 6329 — the one *standardized* deterministic equal-cost tie-breaking scheme (lexicographic path IDs, lowest wins) — precedent for §3.3, including what v1 *avoids* (full path-ID canonicalization; the precision note's rejected cost).
- Andersen, Chung, Lang — *Local graph partitioning using PageRank vectors* — FOCS 2006 — the complementary budgeted-local-exploration primitive: probability-mass budget, `O(1/(εα))`, hub-diluting, no path witness. Different question ("where does random-walk mass go"), not a rejected alternative.
- Beam search — per-level width cap, incomplete, non-monotone in beam width — the contrast that motivates a *global* budget with an optimality certificate at admission.
- **The strongest "this already exists" hits, and the rebuttal** (survey of official docs, 2026-07-10; scoped to: Neo4j GDS/APOC, Memgraph+MAGE, Ladybug, FalkorDB, NetworkX, igraph, TigerGraph): APOC `path.subgraphNodes` (start-node *list*, `limit` = "paths to the n nearest nodes", `NODE_GLOBAL` uniqueness) is multi-seed node-budgeted — and strictly **unweighted hop-nearest**, with no cost/parent output and no documented tie order. NetworkX `multi_source_dijkstra(cutoff=)` is weighted multi-seed with cost+path output — and has **no node-count budget parameter at all**. FalkorDB `algo.SSpaths` (weightProp + costProp/maxCost/maxLen/pathCount) is the richest bounded-weighted machinery surveyed — and is a single-source *path enumerator* whose `pathCount` caps returned paths, not admitted frontier nodes. Neo4j `gds.kSpanningTree` returns exactly-k weighted nodes — single-seed, NP-hard post-hoc heuristic pruning of a full tree, no optimality or determinism. Ladybug ships a genuine **weighted SSSP kernel in core Cypher** — `*WSHORTEST(weightProp)`, inherited from Kùzu 0.8.0 — which computes a hop-bounded single-source shortest-path tree to *all* destinations over non-negative rel-property weights (its algo extension is analytics-only: WCC/SCC/PageRank/k-core/Louvain, plus a Ladybug-added weighted-Kruskal `spanning_forest`). What it has no form of is the **budget**: the kernel is a level-synchronous Bellman-Ford frontier with no priority queue, so there is no cheapest-first settle order to stop, and `LIMIT` truncates output rows only *after* the full hop-ball is explored. The gap is structural, not a missing parameter — a distinct-node budget is not expressible on that algorithm without replacing it. (The trade runs both ways, and it is worth stating honestly: Bellman-Ford makes hop-bounds exact for free, where §8's cheapest-first Dijkstra pays for them with layered `(node, hops)` labels.) **No surveyed system documents the conjunction** — weighted expansion × merged multi-seed frontier × global distinct-node cheapest-first budget × deterministic tie-breaking × per-node cost+parent output — **and none documents deterministic tie-breaking for any comparable operator.** The claim is scoped to the named systems, not a universal negative.
- GraphRAG practice (Microsoft GraphRAG local search; Zep/Graphiti BFS + node-distance reranker; LlamaIndex depth-2 KG retrievers; FalkorDB GraphRAG SDK): hop-bounded collection → post-hoc rerank → token/count truncation, uniformly. The instructive difference: Zep's node-distance *reranker* sorts an already-fetched list by graph distance; this rule makes distance drive *admission* — the expansion and the budget become the same deterministic operation.
